# 💡 Idea Drop Frontend

IdeaDrop Frontend is the web interface of a complete application for creating and managing ideas. This repository contains exclusively the frontend layer, responsible for user interaction and consumption of a separately developed REST API.

The application allows users to view, create, edit, and delete ideas, offering a fluid and responsive experience. Although this repository focuses only on the frontend, the complete project includes a backend and database for data persistence.

![Idea Drop Preview](./public/Idea Drop (1).png)

## 🛠️ Technologies Used

- React
- Vite
- TanStack Router
- TanStack Query
- Axios

## ⚙️ Features

- Page navigation with modern routing
- REST API consumption to retrieve data
- Idea Listing
- Creation of new ideas
- Editing of existing ideas
- Deletion of ideas
- Error Page (Not Found)
- Reusable components
- Structured layout with Header and main pages

## 🧠 Applied Concepts

- Data fetching and cahing with TanStack Query
- Mutations for write operations(create, update, delete)
- Use of loaders with TanStack Router
- Frontend-Backend integration via HTTP
- Scalable code organization
- Proxy configuration with vite

In this project, I deepened my knowledge of TanStack Router and TanStack Query, understanding in practice how to structure a modern frontend application.

With TanStack Router, I learned to create organized routes and use loaders to load data before rendering pages, improving the user experience.

With TanStack Query, I learned to manage asynchronous data efficiently, using automatic caching, API synchronization, and mutations to create, update, and delete information without complication.

These tools showed me a more scalable and performant way to handle data on the frontend.
